# lhci計測Templateの共通化に関する調査メモ

## 背景

複数のフロントアプリでLighthouseをCI/CD実行したい。
そのため計測用のpipeline-templateを共通化したいが、アプリ毎の差分が多く単純な使い回しが難しい。
持ち込みを想定した場合、パッケージマネージャの差分、認証・モックサーバーの有無など、実行条件の違いが問題になる。

---

## この調査の目的

複数のフロントエンドアプリで使い回せるLighthouse用pipeline-templateを作るために以下を明らかにする。

1. どこまで共通化できるか
2. 何がアプリ差分として残るか
3. 差分を.ymlで吸収すべきか、設定ファイルや変数に逃がすべきか
4. ユースケース毎に個別YAMLが必要か
5. 自動生成や補助ツールでどこまで省力化できるか

---

## 現時点の課題感

- パッケージマネージャ差分がある
- ESM/CJSの差分がある
- 認証周りの差分がある
- バックエンドAPI依存の有無がある
- モックサーバー起動方法も考慮が必要
- モックを動かす環境が揃っていない可能性がある

---

## 調査のテーマ[WIP]

### 1. どこまで共通化できるか

- 完全共通の.ymlで対応できるか
- 共通template + 各アプリ毎に設定ファイルを用意する方式
- ユースケース毎に.ymlを分けるべきか

### 2. アプリ毎の差分要因は何か

想定差分は以下のとおり。

#### 実行環境の差分

- Nodeバージョン
- パッケージマネージャ(`npm`, `yarn`, `pnpm`, `bun`)
- ESM/CJS
- 利用ポートNo

#### アプリ構成の差分

- SPA / SSR / SSG
- buildコマンド
- startコマンド
- base path

#### 外部依存の差分

- バックエンドAPI依存の有無
- モックサーバーの有無
- 認証の有無

#### 計測条件の差分

- mobile/desktop
- 実行回数
- assertionの閾値

### 3. 差分をどこで吸収するか

差分を次のどこで吸収するのが適切か整理したい。

#### .yml側で吸収

例:

- 共通のstage
- install/collect/assert/upload手順
- artifact保存
- rules
- cache

#### 変数で吸収

例:

- `APP_BUILD_CMD`
- `APP_START_CMD`
- `APP_BASE_URL`
- `LHCI_TARGET_URLS`
- `LHCI_ASSERT_PROFILE`

#### 設定ファイルで吸収

例:

- `lighthouserc.*`
- app毎のconfig file
- 認証やモック起動条件の外出し

#### 吸収せず、構成パターン毎に分ける

例:

- 静的サイト・認証なし・API依存なし
- 認証あり・API依存あり
- API依存あり・モックサーバー必須
- 認証あり・API依存あり・モックサーバー必須

### 4. 構成差分をオプションとして組み合わせられるか

全案件を固定的な構成パターンに分類するのではなく、
認証、API依存、モックサーバー、HTTPS などの差分要素を
オプション的に組み合わせられる構成にできるかを検討する。

差分要素の例:

- 認証要否
- API依存有無
- モックサーバー要否
- 静的配信/サーバー起動要否
- 特定環境依存有無

検討したいこと:

- 共通テンプレートに対して、差分要素をオプションとして追加できるか
- オプションの組み合わせで対応できる範囲はどこまでか
- 組み合わせが複雑になりすぎる場合、構成パターン毎にテンプレートを分けるべきか

## 現時点の仮説

現状の感触としては完全共通の1本.ymlよりも
共通template+各アプリ設定方式のほうが現実的と思われる。

理由:

- build/startの差分が大きい
- 認証・モック・API依存を1本で吸収すると複雑化しそう
- Lighthouseの対象URLや閾値もアプリ毎に違う
- 実行手順は共通化できても、計測条件までは共通化しにくい　

---

## 整理方針（案）[WIP]

### 共通templateで持つ

- stage
- install
- Lighthouse実行基本手順
- artifact保存
- rules
- cache
- 共通変数インターフェース

### 各アプリで持つ

- buildコマンド
- startコマンド
- 対象URL
- readiness条件
- assertion閾値
- 認証要否
- モック要否
- API接続方法

### 構成次第でテンプレを分ける?

- 静的サイト向け
- API依存あり向け
- 認証あり向け
- モック必須向け
